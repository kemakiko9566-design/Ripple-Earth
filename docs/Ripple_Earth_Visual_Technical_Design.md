# Ripple Earth — 视觉与技术设计

> 叙事驱动。开源优先。比赛级视觉。

---

## 总览

### 叙事流程

`
[Open] 3D Earth (Pacific) ─── 你熟悉的世界
   │
[01] 小黄鸭 ─── 一个你不知道的故事 → 改变你看海洋的方式
   │   ↑ 锚点：所有后续故事从这里延伸
[02] 沙子在消失 ─── 从海洋到陆地，流动的资源
   │   ↑ 沙子像鸭子一样，被洋流和河流搬运
[03] 改变生物钟的灯光 ─── 光在重写自然的日历
   │   ↑ 从可以看见的光，到看不见的影响
[04] 压舱水的远征 ─── 航运是所有这些故事的隐性血管
   │   ↑ 船底水箱在运输整个世界
[Close] Look again. ─── 你再也无法用以前的眼光看它
`

### 设计语言

| 属性 | 值 |
|------|-----|
| 色板 | 深黑 #050810，青色 #00E5FF，橙色信号 #FF6B35，白 gba(255,255,255,0.85) |
| 字体 | Inter (UI) + JetBrains Mono (Data) |
| 网格 | 极细线框 0.5-1px，15°间隔 |
| 运动 | 慢 → 快（scroll-driven ease-out），缓动 cubic-bezier(0.16, 1, 0.3, 1) |

### 技术栈抉择

| 能力 | 选型 | 原因 |
|------|------|------|
| 3D 渲染 | @react-three/fiber + drei | 与现有代码一致，React 生态，Awwwards 标准 |
| 滚动叙事 | GSAP ScrollTrigger + Lenis | 行业标准，Awwwards 大量使用 |
| 数据可视化 | D3.js (选择性) + Canvas 2D | D3 用于复杂图表，Canvas 用于性能敏感粒子 |
| 地图叠加 | globe.gl (vasturiano) — 可选 | 开源、成熟、可直接覆在 Three.js 地球 |
| 平滑滚动 | Lenis | 与 GSAP 深度兼容，流畅感关键 |
| 粒子/动效 | Canvas 2D + GSAP | 避免 Three.js 粒子系统开销，精度可控 |

### [01] 小黄鸭 — 锚点故事

**叙事目标**：让人"啊？"一声，建立"再看一眼"的基调

**数据层**：
- 1992.1.10，北太平洋（44.7°N, 178.1°E），货柜船 Evergreen
- 28,800 只橡皮鸭落海
- 漂移路径：北太平洋 → 印尼 → 澳洲 → 南美 → 北极 → 大西洋
- 时间跨度：15 年

**视觉方案**：

| 元素 | 实现方式 | 开源方案 |
|------|----------|----------|
| 地球 | 同开场（淡入淡出过渡） | Three.js（已有） |
| 路径线 | 细白线 + 黄色已行段 | Canvas 2D setLineDash + dashOffset 动画 |
| 起/终点标记 | 脉冲圆点 + 数据标签 | Canvas 2D rc + illText |
| 年份标注 | 沿线关键节点的 | Canvas 2D illText |
| 过渡 | 地球淡出 → 线框地图出现（已有） | GSAP ScrollTrigger |

关键视觉决定：
- **去掉了船图标和鸭子 emoji**（太卡通，不高级）
- **只用线条 + 数据标注**（NASA 数据可视化风格）
- 路径线从起点脉冲式延伸，走过的变黄色
- D3.js 投影：等距圆柱投影（已有）

### [02] 沙子在消失

**叙事目标**：从"沙子"这个日常东西，引出"现代文明的物质基础在消失"

**数据触发点**：
- 全球每年消耗 500 亿吨沙子
- 沙子是地球上第二大消耗资源（仅次于水）
- 沙漠沙不能用（太平滑）
- 河砂和海底砂开采速度是自然补充速度的 2-3 倍
- 1000 吨沙子 = 1 栋中型建筑 ≈ 5,000 块芯片

**视觉方案**：

| 视觉阶段 | 内容 | 技术 |
|----------|------|------|
| 1. 进沙 | 地面视角：散落的沙粒 → 镜头拉远 → 变成城市 | D3.js / Canvas 粒子系统 |
| 2. 概念可视化 | 每个建筑/芯片底部标注"X 吨沙" | Canvas 2D + GSAP text reveal |
| 3. 全球视角 | 显示全球沙子开采热点 + 河流输送 | Canvas + 地图覆层 |
| 4. 减少 | 沙量以吨/秒为单位快速减少 | D3.js 动画柱状/数字变化 |
| 5. 连接 | 过渡到下一场景：光 | GSAP ScrollTrigger 场景过渡 |

**开源方案**：

| 需实现 | 推荐库 | 理由 |
|--------|--------|------|
| 粒子系统 | Canvas 2D | 简单可控，无需额外依赖 |
| 数字动画 | GSAP nimate | 已引入，平滑数字变化 |
| 柱状/数据图 | D3.js 或 Canvas 手绘 | D3 图表复杂但美观，Canvas 性能更好 |
| 地图坐标 | D3-geo 或 globe.gl | D3-geo 轻量，globe.gl 适合 3D |

每种方案比较：
- **D3.js 柱状图**：成熟，美观，但需要额外引入 D3（≈50KB）
- **Canvas 2D 自制**：无依赖，风格可控，适合科学界面审美
- **globe.gl 热力图**：开源，可直接在 Three.js 地球叠加数据层

推荐：**Canvas 2D 为主 + D3-geo 投影辅助**，保持无额外依赖，风格统一。

### [03] 改变生物钟的灯光

**叙事目标**：从"你开灯"到"灯改变了整个自然界的日历"

**数据触发点**：
- 25 年来，人造夜光增加了 50%
- 每年数十亿只候鸟因灯光迷失方向，撞上建筑
- 城市路灯使树木在春季发芽提前 7-10 天
- 昆虫、鸟类、植物的生物节律被重写
- 40% 的昆虫物种因光污染减少

**视觉方案**：

| 视觉阶段 | 内容 | 技术 |
|----------|------|------|
| 1. 进光 | 黑暗 → 一盏灯亮起 → 更多灯 → 城市 | Canvas 2D 光晕扩散动画 |
| 2. 卫星夜景 | NASA Black Marble 地球夜景图像 | Three.js 纹理替换 / Canvas 覆层 |
| 3. 生命被影响 | 鸟撞建筑（粒子坠落）、树芽提前（时间线） | Canvas 2D 粒子 + GSAP 动画 |
| 4. 昆虫下降 | 夜晚灯光处多虫 → 但整体物种在下降 | D3.js 折线/柱状 + 数字 |
| 5. 连接过渡 | 灯光聚焦到港口/船舶灯光 → 引出下一场景 | GSAP |

**开源方案**：

| 需实现 | 推荐库 | 理由 |
|--------|--------|------|
| NASA 夜景图 | 预加载纹理 
ight_lights.jpg | 免费，高分辨率，直接用于 Three.js 球 |
| 灯光扩散动画 | Canvas 2D + GSAP | Canvas 径向渐变 + GSAP 动画控制 |
| 鸟群/粒子 | Canvas 2D | 简单粒子 + 抛物线路径 |
| 数据图表 | D3.js 或 Canvas 手绘 | 推荐 Canvas（风格统一） |
| 光晕/辉光 | Three.js UnrealBloomPass 或 Canvas | Three.js bloom 效果惊艳但性能开销大 |

关键视觉决定：
- **使用 NASA Black Marble 图层**：项目已有 NASA 主题，自然扩展
- **夜景叠加在 2D 线框地球上**：保持视觉连续性
- 灯光扩散动画用 Canvas 径向渐变：性能好，视觉效果简单但有力

### [04] 压舱水的远征

**叙事目标**：从一个船的底部，引出整个生态系统的重塑

**数据触发点**：
- 一艘船可携带数万吨压舱水
- 每天有 7,000 个物种通过压舱水被跨国运输
- 斑马贻贝（北美五大湖）→ 30 年造成 50 亿美元的生态和经济损失
- 狮子鱼（加勒比海）→ 25 年内扩散至整个东海岸
- 全球 80% 的贸易通过海运

**视觉方案**：

| 视觉阶段 | 内容 | 技术 |
|----------|------|------|
| 1. 进船 | 俯瞰货船 → 切开看到压舱水箱 → 水在交换 | Canvas 2D 剖面图 + 流动粒子 |
| 2. 路径 | 世界地图上航线闪烁 → 每个港口标记水交换 | Canvas / D3-geo 地图覆层 |
| 3. 扩散 | 入侵物种从各港口向外扩散（圆形波纹） | Canvas 2D 波纹扩散动画 |
| 4. 连接图示 | 斑马贻贝、狮子鱼等标志性物种的时间线 | Canvas 时间轴 + 动画 |
| 5. 全球影响 | 叠加物种扩散区域 → 覆盖全球 | Canvas 热力/覆盖区 |

**开源方案**：

| 需实现 | 推荐库 | 理由 |
|--------|--------|------|
| 船剖面图 | Canvas 2D | 简单几何，无需 3D |
| 航线 + 港口 | D3-geo / globe.gl | globe.gl 的 arc 层天然适合航线 |
| 物种扩散波纹 | Canvas 2D | 径向波纹 + 随时间扩散 |
| 热力覆盖 | Canvas 2D + 高斯模糊 | 简单半径内渐变色 |

关键视觉决定：
- **船的剖面图用简约线框风格**（与整体 NASA 科学可视化一致）
- 物种扩散用**波纹从港口向外扩散**的方式
- 最终所有波纹覆盖地球 → 视觉冲击

### 无额外依赖方案

经评估，**这四个场景不需要额外引入任何 npm 包**。现有技术栈覆盖所有需求：

- @react-three/fiber：3D 地球 + 夜景纹理
- GSAP：滚动驱动、数字动画、场景过渡
- Canvas 2D：粒子、扩散、光晕、图表
- Lenis：平滑滚动（未来选配）
- D3.js 为可选项——仅当需要复杂交互图表时引入
- globe.gl 为可选项——仅当需要叠加全球数据热力时引入

---

## 视觉统一规则

1. **所有数据可视化用 Canvas 2D**：风格一致，性能好，无框架依赖
2. **所有数字动画用 GSAP**：平滑统一
3. **颜色遵守设计系统**：深黑 bg，白色线框，青色强调，橙色信号
4. **信息密度**：每次只展示一个数据点/一句话
5. **滚动速度**：慢滚动 → 更多细节；快滚动 → 仅主数据可见
6. **字体**：数据用 JetBrains Mono，正文用 Inter
